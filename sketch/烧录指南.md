# TuneBar 编译和烧录指南

## 成功编译和烧录的方法

### 方法 1: 使用 VS Code + PlatformIO (推荐)

1. **打开项目**
   - 在 VS Code 中打开 `sketch` 文件夹
   - 确保已安装 PlatformIO 扩展

2. **编译**
   - 点击底部状态栏的 **PlatformIO: Build** (✓) 按钮
   - 或按快捷键 `Ctrl+Alt+B`

3. **烧录**
   - 连接 ESP32-S3 设备到电脑
   - 点击底部状态栏的 **PlatformIO: Upload** (→) 按钮
   - 或按快捷键 `Ctrl+Alt+U`

### 方法 2: 使用命令行烧录 (需要 VS Code 先编译好)

1. **先在 VS Code 中完成编译**

2. **使用 esptool 直接烧录**
   ```powershell
   cd sketch
   python -m esptool --chip esp32s3 --port COM5 --baud 115200 --before default-reset --after hard-reset write-flash -z --flash-mode dio --flash-freq 80m --flash-size 16MB 0x0000 .pio\build\esp32-s3-devkitc1-n16r8\bootloader.bin 0x8000 .pio\build\esp32-s3-devkitc1-n16r8\partitions.bin 0x10000 .pio\build\esp32-s3-devkitc1-n16r8\firmware.bin
   ```
   注意：将 COM5 替换为你实际的串口号

## 关键要点

1. **设置环境变量**：运行前设置 `$env:PLATFORMIO_NO_UV="1"`

2. **串口选择**
   - 先查看可用串口：`python -m serial.tools.list_ports`
   - 找到你的 ESP32-S3 设备对应的 COM 口号

3. **项目配置**
   - 配置文件：`platformio.ini`
   - 目标板：`esp32-s3-devkitc1-n16r8`
   - 框架：`arduino`

## 常见问题

### Q: PlatformIO 报错找不到依赖?
A: 设置 `$env:PLATFORMIO_NO_UV="1"` 后重新尝试

### Q: 烧录失败连接不上?
A: 
1. 检查串口号是否正确
2. 检查设备是否连接好
3. 尝试按一下 ESP32-S3 的 BOOT 按钮再烧录
