4. Data Structures

NasData (Main Data Model)

struct NasData {
    NasSystemInfo   system;       // CPU, Memory, Temp, Uptime
    NasDiskInfo     disks[MAX_DISKS];   // Disk status (up to 6 SATA + 3 M.2)
    NasVolumeInfo   volumes[MAX_VOLUMES]; // RAID volumes
    NasServiceInfo  services[MAX_SERVICES]; // Running services
    NasInterfaceInfo interfaces[MAX_NETWORK_INTERFACES]; // Network interfaces
    FanStatus       fan;          // Fan RPM, PWM, temp
    bool            is_online;    // NAS connection status
};

AppConfig (NVS Configuration)

struct AppConfig {
    // WiFi
    char ssid[33];
    char wifipass[65];
    
    // NAS
    char nas_type[16];    // synology/qnap/truenas/...
    char nas_ip[40];
    uint16_t nas_port;
    char nas_user[32];
    char nas_pass[65];
    bool nas_https;
    
    // Display
    uint8_t rotation_angle;  // 0=0°, 1=90°, 2=180°, 3=270°
    uint8_t brightness;
    bool autodim;
    
    // Hardware
    uint8_t sata_disk_count;
    uint8_t m2_disk_count;
    
    // Fan
    FanConfig fan;
};

6. UI Screens

Overview Screen

Layout: 640×172 landscape

  • Top status bar (35px): NAS name, Time, Network speeds (↑/↓), IP
  • Left: CPU usage meter (90×90), CPU temp meter (90×90)
  • Right: Memory bar (72%), Disk bar (50%)
  • Bottom: 6 HDD indicators (LED + label)

Storage Screen

  • Volume list with RAID status
  • Disk details: temp, health, usage
  • Read/write speed indicators

Fan Control Screen

  • Current fan RPM and PWM
  • Auto/manual mode toggle
  • Fan curve visualization
  • Manual PWM slider

Settings Screen

  • NAS type selection
  • Network configuration
  • Display settings (brightness, rotation)
  • Disk configuration (SATA/M.2 count)

CPU Cores Screen

  • Per-core usage bars
  • Load average
  • System uptime

WiFi Screen

  • SSID list with signal strength
  • Connection status
  • IP address display

9. Mock Data Generator

The mock_client.cpp generates realistic simulated data for development and testing:

  • CPU: 20-85% random variation
  • Memory: 40-80% baseline
  • Network: 0-50 MB/s random spikes
  • Disks: 30-90% usage, random health states
  • Fan: 1200-2500 RPM based on "temp"
