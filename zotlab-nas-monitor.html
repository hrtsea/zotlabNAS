1. Demo 简介
是什么

ZotLab NAS Monitor 是一个 ESP32-S3 嵌入式 NAS 监控屏幕的交互式 Web Demo，模拟屏幕根据 NAS 系统 API 实时显示 CPU/内存、温度、存储使用率、硬盘健康度、网络速率等核心监控数据。

面向谁

家庭/小型办公室 NAS 用户、服务器运维人员、数码爱好者，也适用于多 NAS 系统统一监控场景。

主要功能
  • 四大视图切换：总览（圆盘仪表+进度条）、存储（双列8盘布局）、设置（Tab标签页）、CPU监控
  • 实时数据模拟：CPU 温度圆弧仪表、内存/磁盘进度条、上下行网络速率
  • 硬盘状态指示灯：HDD1-6 + M.2-1/2 在线/离线/告警状态一目了然
  • 支持 11 种 NAS 系统：Synology DSM / QNAP QTS / TrueNAS / FNOS / Unraid / Netdata / SNMP / Linux HTTP/Serial / Windows / Mock
2. Demo 创作思路
灵感来源

看到越来越多的家庭和小型办公室部署 NAS 存储数据，但监控 NAS 状态需要打开电脑浏览器或手机 App，交互繁琐且无法固定展示。想要一块百元级的 ESP32-S3 屏幕，像汽车仪表盘一样，实时展示 NAS 的"健康状态"。

想解决的问题

NAS 监控场景下的四大痛点——

  • 交互繁琐：查看状态需要打开手机 App 或电脑浏览器，步骤多、耗时长
  • 设备依赖：必须依赖手机/电脑才能监控，临时查看不方便
  • 品牌封闭：各 NAS 品牌监控方案互不兼容，多系统用户缺乏统一界面
  • 功耗浪费：为查看状态需要唤醒整个电脑，功耗高、不环保
为什么做这个方向

专用 NAS 监控屏幕价格昂贵且功能单一。通过一块 WaveShare 3.49" IPS 屏幕（百元级），配合开源固件，同时监控 QNAP、Synology、TrueNAS、Unraid 等多种 NAS 系统，实现统一、可定制、低功耗的硬件监控方案。

4. TRAE 实践过程
  • 在 TRAE IDE 中描述需求，AI 生成初始创意提案 HTML 页面
  • 在 PlatformIO 项目中逐步迭代，添加 LVGL UI 驱动和网络客户端

  • 基于 SquareLine Studio 设计 UI 原型，导出 LVGL 代码
  • 最终产出可交互的 Web Demo 文件，模拟 640×172 屏幕的真实效果

支持的 NAS 系统
  • Synology DSM：通过 API 获取存储、硬盘健康、CPU/内存、温度数据
  • QNAP QTS：通过 HTTP API 获取存储、RAID 状态、硬盘信息
  • TrueNAS：REST API 集成，监控 ZFS 存储池、告警
  • FNOS (飞牛 NAS)：支持国产 NAS 系统
  • Unraid：Docker API 方式获取数组状态、磁盘信息
  • Netdata：通过 Netdata REST API 获取实时监控数据
  • SNMP：支持标准 SNMP v1/v2c/v3 协议的设备
  • Linux (HTTP)：通过通用 HTTP API 获取 Linux 系统信息
  • Linux (Serial)：通过串口通信获取 Linux 系统数据
  • Windows：通过 SNMP 或 HTTP 获取 Windows 系统信息
  • Mock (Test)：模拟数据源，用于开发和测试
核心功能特性
  • 实时显示：存储容量、RAID 状态、硬盘温度、风扇转速、CPU/内存
  • 多网口监控：支持最多 4 个网络接口（eth0/eth1/wlan0 等）
  • 多设备管理：同时监控多台 NAS，支持快速切换
  • 触控交互：触摸屏操作，滑动切换视图，物理按键控制
  • 风扇控制：PWM 调速、温度曲线、滞回配置、紧急温度阈值
  • 音量控制：ES8311 DAC + ES7210 ADC，支持告警音效
  • OTA 升级：通过网络远程更新固件
  • 屏幕控制：亮度调节、睡眠定时、壁纸设置
6. 技术架构
开发环境
  • IDE：Visual Studio Code + PlatformIO IDE
  • 框架：pioArduino (Arduino Framework for ESP32-S3)
  • UI：LVGL v8.3.11（SquareLine Studio 设计）
  • 文件系统：LittleFS + SD 卡
  • 网络：Wi-Fi Manager + 多协议客户端
目录结构
  • src/：主源码（含 UI、驱动、网络模块）
  • src/ui/：LVGL UI 代码（SquareLine Studio 导出）
  • src/ui/screens/：各页面实现（Overview、Storage、Settings）
  • src/net/：NAS 客户端实现（QNAP/Synology/TrueNAS 等）
  • src/data/：配置管理、统一数据模型
  • src/pcf85063/：RTC 时钟驱动
  • src/tca9554/：IO 扩展器驱动
  • src/es8311/：音频 DAC 驱动
  • src/es7210/：音频 ADC 驱动
  • data/：文件系统数据
  • include/：全局头文件（引脚配置、参数）
统一数据模型

所有数据源填充统一数据结构，UI 仅消费此结构：

  • NasSystemInfo：主机名、型号、运行时间、CPU/内存、温度
  • NasDiskInfo：硬盘名称、型号、温度、健康状态、容量、读写速度
  • NasVolumeInfo：存储卷、RAID 类型、使用率
  • NasNetworkInfo：网络接口、IP 地址、上下行速率
  • FanStatus：风扇转速、PWM、启用状态
网络客户端
  • synology_client：Synology DSM API
  • qnap_client：QNAP QTS API
  • truenas_client：TrueNAS REST API
  • fnos_client：飞牛 NAS API
  • unraid_client：Unraid Docker API
  • netdata_client：Netdata 轮询
  • snmp_client：SNMP v1/v2c/v3
  • serial_client：串口通信协议
  • mock_client：模拟数据（开发测试用）
